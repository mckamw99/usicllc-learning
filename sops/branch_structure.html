Branch Structure

How USIC Academy organizes users, scopes access, and targets learning content through Docebo's 5-level branch hierarchy — and how that hierarchy is maintained.

Overview

01

In Docebo, a branch is an organizational grouping that mirrors USIC's real-world operational structure. Every user belongs to exactly one branch. Branches determine how users are grouped, what content they see, and what Power Users can manage.

The branch hierarchy is sourced from the SharePoint Learning Reference Table maintained by the LAT team. Changes to the org structure originate there and flow into Docebo through automated sync scripts. Power Users do not create or delete branches — that is an LAT function.
5 levels deep
Company → Division → Region → District Full → Cost Center ID. Every user sits at the Cost Center level (level 5).
Cascading access
When a Power User is scoped to a branch, they can see and manage all users and content at that branch and every branch below it.
Enrollment targeting
Learning plans and courses can be targeted at any branch level. An assignment cascades down to all users within that branch's subtree.
SharePoint-driven
Organizational changes (new cost centers, district restructures) are first made in SharePoint and then synced to Docebo using LAT scripts.

The 5-Level Hierarchy

02

Click any level to expand its details, description, and governance notes.

1
Level 1 Company USIC (root)

The single root node that anchors the entire hierarchy. Every user, learning object, and branch path in USIC Academy ultimately traces back here. This node is never modified by Power Users.

  • Named USIC — there is only one Company-level node.
  • Managed exclusively by the LAT team; no Power User has write access at this level.
  • Assigning content or enrollment plans here would affect every single USIC employee.
2
Level 2 Division e.g., East, West, Central

Major operational divisions within USIC. Power Users scoped at Division level have visibility across every region, district, and cost center within that division — a very broad scope that is granted sparingly.

  • Division names are sourced directly from the SharePoint Learning Reference Table.
  • Division-scoped Power Users are uncommon; most managers are scoped lower.
  • Enrollment plans assigned at Division level cascade to all users below — use with care.
3
Level 3 Region e.g., Southeast, Pacific Northwest

Geographic or operational groupings within a Division. Senior Training Delivery Managers (TDMs) may be scoped at the Region level when they need oversight across multiple districts.

  • Regions contain one or more District Full branches beneath them.
  • Region-level enrollment plan assignments cascade to all districts and cost centers.
  • Region names follow the naming convention established in the SharePoint Learning Reference Table.
  • Region-scoped Power Users can view — but not necessarily edit — all users within the region.
4
Level 4 District Full e.g., FL-East District

Individual districts within a Region. This is the most common scope for Training Delivery Managers and Instructors, giving them management authority over all cost centers (crews) within their district.

  • District names follow the format: [State Abbr]-[Direction] District (e.g., FL-East District).
  • The primary management unit for front-line supervisors and field TDMs.
  • Most TDMs and Instructors are scoped to one or more District Full branches.
  • The bulk_move_branches.py script can reassign users across districts in bulk.
5
Level 5 Cost Center ID e.g., 12345

The leaf-level branch — the most granular unit available. All USIC employees are assigned here, with the cost center ID matching USIC's payroll and HR system codes. This is where individual crews live in Docebo.

  • All employees are placed at this level — never at a parent level.
  • Cost Center IDs match USIC HR/payroll system codes exactly.
  • sharepoint_to_branch_moves.py tracks moves at this level when the SharePoint table changes.
  • Terminated users remain at their last Cost Center until processed by archive_terminated.py and cleanup_terminated.py.
  • A Power User scoped here can only see users within that single cost center.

Access & Scope

03

USIC Academy has three Power User profiles. Each Power User is assigned a branch scope that determines which users and content they can manage. Access always cascades downward — a user scoped to a Region can see all Districts and Cost Centers below it, but nothing above or beside it.

Principle of least privilege: When setting up a Power User, assign the lowest branch level that still lets them do their job. A district-level TDM does not need Region-level scope.

Enrollment Targeting

04

Learning plans and courses in USIC Academy can be targeted to any branch in the hierarchy. The assignment cascades automatically to all users within that branch's subtree — you do not need to assign users individually.

Important: Adding a course to an already-published learning plan auto-enrolls all current members of that plan immediately. Test in a draft plan first when rolling out new content to a large branch.
Assigning at Company (Level 1)
Every USIC employee is enrolled. Reserved for mandatory compliance content that applies to the full workforce.
Use with caution
Assigning at Division (Level 2)
Targets an entire division. Good for division-specific policy or role-based content that applies broadly.
Broad reach
Assigning at Region / District (Levels 3–4)
The most common use case — targets a specific operational area. TDMs manage enrollments at this level most frequently.
Recommended scope
Assigning at Cost Center (Level 5)
Targets a single crew. Useful for crew-specific remediation or onboarding content for a new cost center.
Granular control

Branch Maintenance

05

Branch maintenance is an LAT function. The scripts below automate the most common maintenance tasks. Power Users do not run these directly — they submit requests to the LAT team who execute them.

sync_branches.py

Reads the SharePoint Learning Reference Table and creates or updates branches in Docebo to match the current org structure.

sharepoint_to_branch_moves.py

Detects when users have changed cost centers in SharePoint and moves those users to their correct Docebo branch.

bulk_move_branches.py

Batch-moves a list of users from one branch to another. Used for district restructures or large group transfers.

pull_branches.py

Exports the current Docebo branch tree to a local file for auditing, comparison, or troubleshooting.

archive_terminated.py

Identifies terminated users (via field_17 Termination Date and field_20 Status) and moves them to the archive branch. Runs on a scheduled task.

cleanup_terminated.py

Follows up on archived terminated users — deactivates accounts and removes enrollments as appropriate per retention policy.

To request a branch move for a user or crew, contact the LAT team via Zendesk. Include the employee's name, current branch, and destination branch.
