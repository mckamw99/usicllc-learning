Generating Reports in Docebo

A complete walkthrough for Superadmins and Power Users — how to build a custom report, choose the right report type for the question you’re answering, and set visibility so the right people can see it.

01

Overview

Reports in Docebo let you extract and analyze platform data — user progress, course completions, learning-plan status, ILT attendance, certifications, and more. Both Superadmins and Power Users (with the appropriate Reports permission) can build them.

Two report areas

AreaWhat it is & when to use it
Reports
Custom reports builder — recommended		The modern report builder (named Reports since Feb 4, 2026; formerly “New reports”). Create, configure, schedule, and preview a report from tabs on a single page. Covers the vast majority of report types. Use this by default.
Legacy reports
formerly “Reports”		The older module (renamed Legacy reports on Feb 4, 2026). Use only for a report type not yet available in the Reports builder. See Legacy Reports below.

Both areas live in the left sidebar under Data & analytics (the chart icon) — there is no longer a top-right gear / Admin menu.

Data is not real-time. The data in any report is refreshed automatically on a periodic schedule, not the instant you open it. Recent activity (last few hours) may not yet appear.

Choosing the Right Report Type

Every report is built around a data pairing — for example Users – Courses pairs people with their course progress. The fastest way to pick a type is to start from the question you’re answering, not the columns you want.

03

Creating a Custom Report

  1. 1
    Open the Reports areaIn the left sidebar, hover over the Data & analytics (chart) icon and select Reports.
  2. 2
    Start a new reportOn the Custom reports builder tab, click the + (plus) icon in the top-right corner. The Create a new custom report panel opens.
  3. 3
    Select the report TypeIn the first step (Type), choose the report type using the decision guide above. Confirm to open the report’s configuration page.
  4. 4
    Set PropertiesOn the Properties tab, enter a clear Title and Description, then set the Visibility — see Setting Visibility next. This is where you make the report viewable to Power Users.
  5. 5
    Choose FiltersOn the Filters tab, select the users, groups, or branches and the items (courses, learning plans, sessions, etc.) the report should cover.
  6. 6
    Pick Report Fields (columns)On the Report fields tab, select the data columns to display. Use View options for additional or sensitive fields, and Sorting options to order the output.
  7. 7
    (Optional) Schedule itOn the Scheduling tab, send the report on a recurring basis — daily, weekly, monthly, quarterly, bi-annually, or annually — to chosen recipients.
  8. 8
    Preview, save, and exportUse Preview to verify the output, then save. To export, use the report’s ellipsis (…) menu and choose CSV or XLS. Exports run as a background job; you’re notified by email and in-platform when ready.
Heads-up: Export requires the Notifications app to be active. A Power User running a report only sees data for the courses, users, and branches they have visibility over — Superadmins see everything.
04

Setting Visibility — Superadmins & Power Users

Standard for USIC Academy: so that the right people can find a report, set its Visibility (on the Properties tab) to “All Superadmins and Power Users.” Reports left at the default may be visible only to Superadmins and the creator.

In the Visibility area of the Properties tab, choose one of three rules. These control who sees the report listed on the Reports page:

All Superadmins
Visible to every Superadmin (and the creator) only. Power Users cannot see it, apart from the one who created it.
All Superadmins and Power Users
Visible to every Superadmin and to all Power Users who have report permissions. This is the setting that lets your team find and run the report.
All Superadmins and the selected Power Users
Visible to Superadmins, the creator, and only the Power Users you choose — named individually, or by group / branch. Use when a report should reach a limited audience.
Permission still required. Visibility alone isn’t enough — a Power User also needs the Reports › View permission (in their Power User profile) to open the Reports page at all. Adding Reports › Edit lets them create and duplicate reports. The creator of a report always keeps visibility on it, regardless of the rule.
05

Legacy Reports (When Needed)

Use Legacy reports only for a report type that isn’t yet available in the Reports builder. The area was renamed from “Reports” to Legacy reports on Feb 4, 2026.

  1. 1
    Open Legacy reportsIn the left sidebar, hover over the Data & analytics (chart) icon and select Legacy reports.
  2. 2
    Create the reportClick Create custom report, enter a Title, choose the report type, then select the Users, Group, or Branch to run it on.
  3. 3
    Set visibilityClick the entry in the report’s Filter column and choose All Superadmins and Power Users (or All Superadmins, or Select Superadmins and Power Users), then Confirm.
Migrating: Existing legacy reports can be moved to the Reports menu — properties, visibility, filters, and fields carry over (schedules do not). Run the migration from the Reports page.
06

Best Practices & Tips

  • Name reports for discovery. Use a consistent prefix so the team can scan the list — e.g. [ULE] Course Completion — Monthly.
  • Default visibility to “All Superadmins and Power Users” unless the report contains a limited audience’s data — then use the selected-Power-Users option.
  • Separate branch and group reports. It’s recommended to build distinct reports for branches vs. groups to make membership easier to identify.
  • Remember data isn’t live. If today’s activity is missing, it likely hasn’t hit the periodic refresh yet — re-run later rather than rebuilding.
  • Protect sensitive fields. Invisible / additional fields can be included for Superadmins and Power Users with the Can see invisible fields permission.
  • Schedule recurring needs instead of re-running manually — the report is delivered to recipients on the cadence you set.
