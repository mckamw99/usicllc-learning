Role Overview

01

The Learning Manager is a Power User role assigned to regional learning professionals responsible for managing training administration within their assigned branches. Training Delivery Managers (TDMs) have significantly broader access than Instructors and are the primary LMS administrators at the regional level.

TDMs are responsible for enrollment management, learning plan administration, ILT session creation, reporting, and compliance tracking for learners in their assigned region.

Permissions

✓ You CAN
  • Enroll, unenroll, and archive user enrollments in courses and learning plans
  • Create and manage ILT sessions
  • Mark and correct attendance for sessions in your branches
  • Create, edit, and publish learning plans
  • Run, edit, and export compliance and completion reports
  • Manage Instructor Power Users within your branches
  • Add users to branches within your assignment
  • Send enrollment notifications to learners
✗ You CANNOT
  • Create or upload new course content (SCORM/video)
  • Modify core course settings (passing score, attempts)
  • Access users or branches outside your assignment
  • Edit platform configuration or system settings
  • Create or modify Power User profiles
  • Access financial or e-commerce features
  • Permanently delete courses or learning plans
  • Modify system-level integrations or SSO settings

Managing Enrollments

04

Enrollment management is one of the most frequent TDM tasks. You can enroll learners individually, in bulk, or via learning plan auto-enrollment rules.

4.1 — Enrolling a User in a Course
  1. 1
    Go to People › UsersNavigate to the Users section and search for the learner by name or employee number.
  2. 2
    Open the user profileClick the learner's name to open their profile.
  3. 3
    Navigate to the Courses tabClick the Courses tab on the user profile.
  4. 4
    Click EnrollClick the Enroll button and search for the course by name or course code. Select the course and confirm.
  5. 5
    Set enrollment optionsIf applicable, set a due date, enrollment status, or level. Click Save to confirm the enrollment.
4.2 — Enrolling a User in a Learning Plan
  1. 1
    Go to People › UsersSearch for and open the learner's profile.
  2. 2
    Navigate to the Learning Plans tabClick the Learning Plans tab on the user profile.
  3. 3
    Click Enroll in Learning PlanSearch for the learning plan by name or code. Select it and confirm.
  4. 4
    Confirm the enrollmentThe learner will be automatically enrolled in all courses within the learning plan based on its configuration.
Note: For bulk enrollments across many users, use the bulk enrollment tool under Content & Delivery › Learning Plans › [Plan] › Enrollments › Add Users. This is the preferred method for onboarding cohorts.
4.3 — Archiving a User Enrollment
  1. 1
    Open the user profileGo to People › Users, find the learner, and open their profile.
  2. 2
    Navigate to Courses or Learning PlansFind the course or learning plan to archive.
  3. 3
    Archive the enrollmentClick the … menu next to the enrollment and select Archive enrollment. Confirm. The record is retained but removed from the learner's active view.
4.4 — Unenrolling a User
  1. 1
    Open the user profileGo to People › Users, find the learner, and open their profile.
  2. 2
    Navigate to Courses or Learning PlansFind the course or learning plan to remove.
  3. 3
    Remove the enrollmentClick the remove/unenroll option. Confirm the action. Removing a learner from a learning plan does not automatically remove them from individual courses within it.

Managing Learning Plans

05

Learning plans are the primary delivery mechanism for structured training programs at USIC. As a TDM you can create, edit, publish, and manage learning plan enrollments.

5.1 — Creating a New Learning Plan
  1. 1
    Go to Content & Delivery › Learning PlansNavigate to the Learning Plans section of the left sidebar.
  2. 2
    Click New Learning PlanClick the button to create a new learning plan.
  3. 3
    Complete the detailsEnter the title, code (use the established naming convention), description, and thumbnail if applicable. Set the plan type and any completion rules.
  4. 4
    Add coursesClick Add Content to add courses to the plan. Set ordering, mandatory/optional status, and completion dependencies between courses.
  5. 5
    Assign to catalogsAssign the learning plan to the appropriate catalog(s) so it is visible to the right learners and Instructors.
  6. 6
    PublishWhen ready, set the plan status to Published. Unpublished plans are not visible to learners.
Note: Always confirm the naming convention and catalog assignment with Matthew McKay before publishing a new learning plan to avoid duplication or catalog conflicts.
5.2 — Editing an Existing Learning Plan
  1. 1
    Find the learning planGo to Content & Delivery › Learning Plans and search by name or code.
  2. 2
    Click EditOpen the plan and click Edit.
  3. 3
    Make your changesUpdate courses, ordering, descriptions, or catalog assignments as needed.
  4. 4
    Save and confirmSave the plan. Changes to published plans take effect immediately for enrolled learners.
Note: Removing a course from a published learning plan will not remove it from learners already enrolled — it only affects new enrollments going forward.

Creating and Managing ILT Sessions

06

Unlike Instructors — who can only manage their own sessions — TDMs can create new ILT sessions, set locations, assign instructors, and manage rosters.

6.1 — Creating a New ILT Session
  1. 1
    Go to Content & Delivery › CoursesFind the ILT course you want to add a session to. Filter by course type 'Classroom'.
  2. 2
    Open the course and navigate to SessionsClick into the course and select the Sessions tab.
  3. 3
    Click New SessionClick the button to create a new session.
  4. 4
    Complete session detailsEnter the session name, date, start/end time, location (select from the 68 registered USIC locations), and maximum seats.
  5. 5
    Assign an InstructorAssign one or more Instructors to the session from the Power Users in your branch.
  6. 6
    Set enrollment optionsConfigure enrollment policy — open enrollment, manager approval, or admin-only. Set any waitlist rules.
  7. 7
    Save the sessionSave and confirm. The session is now available for enrollment.
6.2 — Managing Session Attendance
  1. 1
    Navigate to the sessionGo to Content & Delivery › Courses › [Course] › Sessions › [Session].
  2. 2
    View the rosterOpen the session to see all enrolled learners and their current attendance status.
  3. 3
    Mark or correct attendanceUpdate attendance status for each learner: Attended, Absent, or Waiting List. As a TDM you can correct attendance after a session has been closed — something Instructors cannot do.
  4. 4
    Mark session completeSet the session status to Complete once all attendance has been recorded.

Running Reports

07

Reporting is a core TDM responsibility. USIC's Docebo instance has over 100,000 active users across 190+ learning plans — accurate reporting is essential for compliance and operations.

Key Report Types

Course Completion Report
Track which learners have completed a specific course or set of courses
Learning Plan Progress Report
See completion status of learners enrolled in a learning plan
ILT Session Attendance Report
View attendance records for a session or range of sessions
User Enrollment Report
See all courses and learning plans a user is enrolled in
Compliance Report
Identify learners who are overdue or non-compliant on required training
Branch/Region Summary
Aggregate completion rates across your assigned branches

Running and Editing Reports

  1. 1
    Go to Data & Analytics › ReportsNavigate to the Reports section from the left sidebar.
  2. 2
    Select or create a reportLook for pre-configured reports. To create a new report, click New Report and select the report type.
  3. 3
    Apply filtersFilter by branch, course, learning plan, date range, or completion status to scope the results to your needs.
  4. 4
    Edit and customizeTDMs can edit assigned reports — add or remove fields, adjust filters, and save customized versions for ongoing use.
  5. 5
    Preview and exportPreview the report results. Export to Excel or CSV for sharing with managers, compliance teams, or district leadership.
  6. 6
    Schedule recurring reportsFor reports you run regularly, use the Schedule option to have Docebo automatically generate and email them on a set cadence.
Note: TDMs can view notifications under Configuration › Automation › Notifications. Creating or editing notifications requires escalation to the Content Creator or LMS Administrator.

When to Escalate to the LMS Administrator

10

TDMs handle most regional LMS administration independently. The following situations require escalation:

A new Power User profile needs to be created or modified
A user needs to be assigned or reassigned to a different branch outside your scope
A course needs to be created, uploaded, or retired from the platform
System-level settings, SSO, or integrations need to be changed
A completion record needs to be manually overridden at the system level
A catalog needs to be created or its structure significantly changed
You are experiencing platform errors that affect multiple users
A bulk data import or user sync needs to be performed

Training Delivery Manager (TDM) Do's and Don'ts

11
✓ DO
  • Verify branch assignments before enrolling users
  • Use bulk enrollment for cohort onboarding
  • Confirm naming conventions before creating new learning plans
  • Correct attendance errors as soon as they are reported
  • Schedule recurring reports for compliance tracking
  • Communicate session changes to affected Instructors and learners
  • Escalate system-level issues to the LMS Administrator promptly
  • Keep the LMS Administrator informed of new programs or catalog changes
✗ DON'T
  • Enroll users in learning plans outside their job role or region
  • Enroll users one-by-one when bulk tools are available
  • Create duplicate learning plans — check if one already exists
  • Leave attendance errors unresolved — they affect compliance data
  • Pull one-off reports when a recurring schedule would be more efficient
  • Make session changes without notifying the assigned Instructor
  • Attempt to work around system errors without logging a ticket
  • Publish new learning plans without confirming catalog alignment
