Role Overview

01

The Content Creator is a Power User role assigned to L&D team members responsible for building, uploading, organizing, and publishing training content. This is the highest-access content management role — Content Creators have ALL access across courses, learning plans, categories, and catalogs.

Content Creators are the architects of the USIC learning content library. They own the full content lifecycle from initial upload through publication and ongoing maintenance.

Permissions

✓ You CAN
  • Create and edit courses
  • Upload and manage SCORM, video, and content files
  • Organize courses into categories and catalogs
  • Create, edit, and publish learning plans
  • Manage course settings (passing score, completion rules)
  • Manage the Central Repository (your assigned folder)
  • Create, edit, and manage notifications
  • View and edit certificates for assigned courses
  • Manage platform menus and pages
✗ You CANNOT
  • Delete courses (you do not have this permission)
  • Enroll or unenroll learners
  • Create or manage ILT sessions
  • Mark attendance for any sessions
  • Access or modify individual learner profiles
  • Reset learner progress or completion records
  • Modify system-level integrations or SSO settings
  • Create or modify Power User profiles
  • Access user management or reporting tools

Course Governance Standards

04

As a Content Creator you are responsible for course standardization, content quality, accessibility, and structure per the USIC LMS Course Governance framework. Every course must meet the following standards before publishing.

4.1 Required Course Fields

FieldRequiredNotes
CodeYFormat: [DEPT]-[PROGRAM]-[TOPIC]-[SEQ]-[VER], max 50 chars
TitleYMust be simple and clear. No jargon or special characters without L&D approval.
Short DescriptionYBrief text shown on the course card — add context beyond the title
DescriptionYDetailed information about what learners will learn. Must be thorough.
CategoryYAssign to the correct USIC category (LOC, EHS, LDR, ULE, RCN, etc.)
ThumbnailYUpload a course thumbnail image. Required for all courses.
Average TimeYSet the estimated completion time. Used in reporting and displayed to learners.
Assigned SkillsY*Up to 15 skills per course. *Confirm with LMS Admin — pending career path confirmation.
Certification (Addl. Field)YIndicate if the course results in a certification
Content Owner (Addl. Field)YMaps to the responsible department
Course Type (Addl. Field)Y if applicableMandatory (requires L&D OPS Approval) / Required / Optional
Course Owner (Addl. Field)YThe individual accountable for this course's content
Launch Date (Addl. Field)Addl.Record the date the course was published.
Note: Course Additional Fields are found in the course Properties tab under the Additional Fields sub-tab.

4.2 Course Code Naming Convention

All USIC course codes must follow this structure (max 50 characters):

[DEPT]-[PROGRAM]-[TOPIC]-[SEQ]-[VER]
SegmentPurposeExampleMax
DEPTOwner/Audience — responsible departmentHR, EHS, ULE5 chars
PROGRAMCurriculum — specific program or initiativeAMP, ULE, ONB5 chars
TOPICContent — subject matterLEAD, SEC, LOC5 chars
SEQSequence — position in the learning path101, 205, 3-A4 chars
VERVersion — tracks major content revisionsV3, V43 chars

4.3 Course Type Values

ValueImplied ObligationNotes
MandatoryCommand from authority — law, official rule, or management mandateRequires L&D OPS Approval before publishing. Non-negotiable.
RequiredNecessary condition — prerequisite or general policyNo OPS approval needed. Consequences for non-completion.
OptionalLearner choice — enrichment or supplemental contentNo negative consequence for skipping.

4.4 Additional Governance Requirements

AccessibilityAll images in course content must include alternative text (alt text) for screen reader compatibility. This is a USIC governance mandate.
SecurityCourse titles must comply with OWASP guidelines. Embedded media must use the pre-approved iFrame allow list — do not embed external content from unlisted sources.
Course LifecycleCourses must be set to Under Maintenance before major edits on a live course. Move to Published only when the course is ready for learners.
SkillsAssign up to 15 relevant skills to each course. Skills link courses to USIC's career path framework. Confirm the correct skills with the LMS Administrator if unsure.
Content QualityTitles must be simple and clear. Descriptions must be detailed and informative. Avoid jargon or ambiguous abbreviations in titles.

Managing Courses

05

Course management is the core function of the Content Creator role. USIC currently has 3,000+ courses across 16 categories.

5.1 — Creating a New Course
  1. 1
    Go to Content & Delivery › CoursesNavigate to the Courses section of the left sidebar.
  2. 2
    Click New CourseSelect the course type: E-Learning (SCORM/xAPI), Classroom (ILT), Webinar, or other.
  3. 3
    Complete the course detailsEnter the course name, code (follow the naming convention), description, category, and language. Upload a thumbnail image.
  4. 4
    Configure course settingsSet passing score, maximum attempts, completion rules, enrollment policy, and any due date or expiration rules.
  5. 5
    Upload contentFor e-learning courses, upload your SCORM or xAPI package, video file, or other content. Verify the content loads and tracks correctly.
  6. 6
    Assign to catalogsAssign the course to the appropriate catalog(s) so the right learners and TDMs can access it.
  7. 7
    Test before publishingPreview the course and verify tracking works correctly. Have a test user complete it if possible before publishing broadly.
  8. 8
    Publish the courseSet the course status to Published. The course is now visible to learners and available for enrollment.
Note: Always confirm the course code and category with your L&D Manager before publishing. Avoid duplicating existing courses — search first.
5.2 — Editing an Existing Course
  1. 1
    Find the courseGo to Content & Delivery › Courses and search by name or course code.
  2. 2
    Set to Under MaintenanceBefore making edits to a live published course, set the status to Under Maintenance first to prevent learners from accessing it mid-edit.
  3. 3
    Click EditOpen the course and click Edit.
  4. 4
    Make your changesUpdate content, settings, descriptions, catalog assignments, or category as needed.
  5. 5
    Replace content if neededTo update the SCORM or video file, navigate to the course content tab and upload the new file. Remove the old version once confirmed working.
  6. 6
    Save, test, and re-publishSave your changes, verify everything works correctly, then set the status back to Published.
5.3 — Archiving a Course
⚠ Important: Once a course is archived in Docebo, it is automatically removed from your Power User resources. You will no longer be able to see, track, or manage it. Complete all necessary tasks before archiving.
  1. 1
    Complete pre-archive tasks firstRemove the course from any active learning plans and catalogs. Notify your TDM so they can update any affected enrollments or reporting.
  2. 2
    Open the courseGo to Content & Delivery › Courses and open the course to be archived.
  3. 3
    Change the category to Archive (ARC)Edit the course and change the category to Archive. Update the course code prefix to ARC if not already done.
  4. 4
    Unpublish the courseSet the course status to Unpublished so it is no longer visible to learners.
  5. 5
    Document the changeNote the archival in your team's content tracking documentation and inform your TDM. Learner completion history is preserved even after archiving.
Note: Do not delete courses — always archive them. Deletion permanently removes completion history and cannot be undone.

Managing Learning Plans

06

Content Creators have full access to create, edit, and publish learning plans. Learning plans bundle courses into structured programs and are the primary way USIC delivers training to specific learner populations.

6.1 — Creating a New Learning Plan
  1. 1
    Go to Content & Delivery › Learning PlansNavigate to the Learning Plans section.
  2. 2
    Click New Learning PlanClick the button to create a new plan.
  3. 3
    Complete the detailsEnter the title, code (use the established convention), description, and thumbnail. Set plan type and any completion rules or due dates.
  4. 4
    Add coursesClick Add Content to add courses to the plan. Set ordering, mandatory vs. optional status, and any completion dependencies.
  5. 5
    Assign to catalogsAssign the learning plan to the appropriate catalog(s) to control visibility.
  6. 6
    PublishWhen ready, set the plan to Published. Coordinate with your TDM before publishing plans that will trigger auto-enrollments.
6.2 — Editing an Existing Learning Plan
  1. 1
    Find the learning planGo to Content & Delivery › Learning Plans and search by name or code.
  2. 2
    Click EditOpen the plan and click Edit.
  3. 3
    Make your changesAdd or remove courses, reorder content, update descriptions, or change catalog assignments.
  4. 4
    SaveSave the plan. Always communicate changes to your TDM before saving.
⚠ Critical: Adding a course to a published learning plan immediately auto-enrolls ALL current plan members into that new course. Confirm with your TDM before adding courses to any active plan with large enrollments. Removing a course does not unenroll current learners — it only affects future enrollments.

Managing Catalogs

07

Catalogs control which courses and learning plans are visible to which learners and Power Users. Content Creators have access to all 49 USIC catalogs.

7.1 — Assigning Content to a Catalog
  1. 1
    Go to Content & Delivery › CatalogsNavigate to the Catalogs section.
  2. 2
    Find the appropriate catalogSearch by name. USIC has 49 catalogs organized by line of business, region, and program type.
  3. 3
    Add contentClick Add Content and search for the course or learning plan to add. Select it and confirm.
  4. 4
    Verify visibilityConfirm the content is now visible to the intended audience by checking the catalog's learner visibility settings.
7.2 — Removing Content from a Catalog
  1. 1
    Open the catalogGo to Content & Delivery › Catalogs and open the relevant catalog.
  2. 2
    Find the contentLocate the course or learning plan to remove.
  3. 3
    Remove itClick the remove option. This does not unenroll learners who are already enrolled — it only prevents new enrollments from this catalog.
  4. 4
    Communicate the changeNotify your TDM of catalog changes that may affect active learner populations.

Central Repository

08

The Central Repository is the shared USIC content library in Docebo. Content Creators have access to 1 assigned repository folder for storing and managing reusable training assets.

Best for
Reusable media assets shared across multiple courses
Supported types
MP4, PDF, Word, images, SCORM packages
  1. 1
    Go to Content & Delivery › Content Center › Central RepositoryNavigate to the Central Repository section of the left sidebar.
  2. 2
    Navigate to your assigned folderOpen your assigned USIC repository folder. You can only create, edit, and delete training materials within your assigned folders.
  3. 3
    Upload your fileClick Upload and select your file. Give it a clear, descriptive name and add relevant tags.
  4. 4
    Use in coursesOnce uploaded, add the asset to any course via the course Training Material tab › Add training material › Central repository. No need to re-upload the same file to multiple courses.
Note: Unpublished Creator lessons will not appear in the Central Repository list.

Notifications & Certificates

09

Content Creators have permissions to create, edit, and manage notifications and certificates — capabilities not available to TDMs or Instructors.

9.1 Managing Notifications

Go to Configuration › Automation › Notifications to view and manage notifications for your assigned courses and learning plans
Create new notifications triggered by: course enrollment, completion, due date approaching, or certificate issuance
Edit existing notifications to update messaging, timing, or recipient rules
⚠ Important: Always test a notification before activating it on a high-enrollment course — coordinate with your TDM to avoid unintended mass emails.

9.2 Managing Certificates

Navigate to Content & Delivery › Courses › [Course] › Properties › Certifications & Retraining to associate a certificate template with a course
Content Creators can view and edit certificate templates for their assigned courses
Certificates are automatically issued to learners upon course completion based on the template settings
Contact the LMS Administrator if a new certificate template needs to be created from scratch
Note: If a course requires a certification with a retraining period, configure the recertification rules in the Certifications & Retraining tab of the course properties.

SCORM & xAPI Tracking

10

One of the most common causes of learners getting stuck In Progress is a mismatch between what the SCORM or xAPI package is reporting and what Docebo is configured to look for.

10.1 How Docebo Tracks SCORM Completion

Docebo marks a SCORM course complete when the package sends a completion signal AND the learner presses the Close/Exit/Submit button. Closing the browser tab without pressing this button will not save completion.

SCORM VersionCompletion FieldValid Values
SCORM 1.2cmi.core.lesson_statuscompleted, passed
SCORM 2004 3rd Ed.cmi.completion_statuscompleted
SCORM 2004 3rd Ed.cmi.success_statuspassed (for pass/fail tracking)
Note: Use SCORM 2004 3rd Edition for new content. SCORM 2004 4th Edition has stability issues with Docebo's completion triggers.

10.2 Authoring Tool Export Settings

ToolSettingRecommended for Docebo
Articulate StorylinePublish › LMS › Reporting tabTrack using: Passed/Incomplete. Sends cmi.core.lesson_status = passed.
Articulate RiseExport › LMS › Tracking › Report completionSet to: Course is complete when learner completes all content. Use SCORM 2004 3rd Edition output.
Both toolsExit/Close buttonAlways include a final completion slide with a visible Close or Exit button. Docebo requires this to commit tracking data.

10.3 Docebo Completion Rule Settings

  1. 1
    Go to the course Training Material tabContent & Delivery › Courses › [Course] › Training Material tab. Find the SCORM item and click hamburger menu › Settings.
  2. 2
    Check the End Object MarkerEnsure 'The course is completed when this training material is completed' is checked if this is the only or primary training material.
  3. 3
    Check the Score calculation methodIf your SCORM includes a test, verify the method matches your intent. If your SCORM sends a score of 0, it will be included in average/sum calculations.
  4. 4
    Verify Completion RulesGo to Course Properties › Completion Rules. If set to All training materials, every item in the course must be completed — including supplemental PDFs or videos.
  5. 5
    Check View mode for mobileConfirm whether the package is set as playable on the mobile app. SCORM packages must be fully self-contained to work in Go.Learn offline mode.

10.4 Troubleshooting In Progress — Diagnostic Checklist

  • 1
    Did the learner press the Close/Exit/Submit button inside the content when finished? If not, ask them to retake and close properly.
  • 2
    Does the SCORM send the correct completion value? Use Docebo's SCORM Debug tool: Content & Delivery › Courses › [Course] › Training Material › [SCORM] hamburger menu › SCORM Debug.
  • 3
    Is the course completion rule set to All training materials? Check whether the learner completed every item — including PDFs, videos, or supplemental files.
  • 4
    For xAPI: Is Track all activities toggled correctly for your package structure?
  • 5
    Did the learner complete the course on the mobile app while offline? Ask them to open Go.Learn while connected to sync.
  • 6
    Did the learner's session time out mid-course? Ask them to retake in a single session.
  • 7
    Are third-party cookies blocked in the learner's browser? SCORM tracking requires cookies. Check Settings › Site Settings › Cookies in Chrome.
  • 8
    If none of the above resolve it: escalate to the LMS Administrator to manually mark the enrollment as Completed.

Creating Course QR Codes for the Mobile App

11

USIC uses Docebo's Go.Learn mobile app for field-based learners. You can create a QR code that, when scanned, opens a specific course directly in Go.Learn using Docebo's deep linking feature.

Note: The learner must have Go.Learn installed and be logged in to usic.training for the deep link to work. Always test before deploying.

Deep Link URL Format

DestinationDeep Link FormatExample
Specific coursegolearn://course/{course_id}golearn://course/1234
Specific channelgolearn://channels/{channel_id}golearn://channels/56
App Store (iOS)https://apps.apple.com/app/go-learn/id1362542319
Google Play (Android)https://play.google.com/store/apps/details?id=com.docebo.eolo.staging

Step-by-Step: Creating a QR Code

  1. 1
    Find the Course ID in DoceboOpen the course in Content & Delivery › Courses and look at the URL in your browser. The course ID is the number in the URL — e.g., in .../learn/course/1234/course-name the ID is 1234.
  2. 2
    Build the deep link URLUse the format: golearn://course/{course_id} — replacing {course_id} with the number from Step 1.
  3. 3
    Open a QR code generatorGo to QR Code Monkey (qrcode-monkey.com) or QR Code Generator (qr-code-generator.com).
  4. 4
    Enter the deep link as the QR contentSelect URL or Text as the input type. Paste your golearn:// deep link into the input field.
  5. 5
    Customize (optional)Add the USIC logo, adjust colors to match USIC branding, and choose high-resolution PNG or SVG output.
  6. 6
    Download and name the fileSave with a descriptive filename — e.g., QR_LocatingElectric_Course1234.png
  7. 7
    Test before deployingUsing a device with Go.Learn installed and logged in to usic.training, scan the QR code. Verify it opens the correct course in the app — not a browser. Test on both iOS and Android if possible.
  8. 8
    Embed in materialsAdd the QR code to printed training materials, onboarding documents, SharePoint pages, or course content. Include instructions: "Scan with your phone camera or Go.Learn app to open this course."
Note: Course deep links only work for courses the learner is enrolled in. If not enrolled, they will see an access error in the app.

Content Creator Do's and Don'ts

14
✓ DO
  • Complete all required fields before publishing — code, description, thumbnail, average time, additional fields
  • Follow the [DEPT]-[PROGRAM]-[TOPIC]-[SEQ]-[VER] course code convention
  • Add alt text to all images in course content
  • Set course status to Under Maintenance before major edits on a live course
  • Assign up to 15 skills per course aligned to the USIC career path framework
  • Search for existing content before creating anything new
  • Test content thoroughly before publishing — preview and verify SCORM tracking
  • Archive retired courses — set to Under Maintenance, then unpublish, then change category to ARC
  • Get L&D OPS Approval before publishing any course with Course Type = Mandatory
✗ DON'T
  • Publish a course with missing required fields
  • Create a course without a correctly formatted code
  • Upload image-based content without accessibility alt text
  • Edit a live published course without first setting it to Under Maintenance
  • Publish a course without assigned skills (confirm with LMS Admin if unsure)
  • Create duplicate courses — always search first
  • Publish untested content — always preview first
  • Attempt to delete courses — you do not have delete permission
  • Publish Mandatory courses without L&D OPS approval
