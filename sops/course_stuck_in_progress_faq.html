Course Stuck in “In Progress”

Why a learner’s course refuses to flip to Completed — how training material settings drive course completion, what breaks it, and exactly how to fix each case in Docebo.

How Course Completion Works

Section 1

The single rule that explains almost every “stuck” course: a course is marked Completed only when every training material that counts toward it is individually marked complete. Course status is calculated up from the material level — so if even one required material never registers as complete, the course stays In Progress, no matter what the learner saw on screen.

There are two important modifiers to that rule:

End Object Marker. If a material is set as an end object marker, completing that one material marks the whole course complete — even if optional materials remain unfinished. A course can have more than one. This is the cleanest way to control completion when a course mixes required and optional content.
Hidden material. Hidden training material is not counted toward completion for learners who never viewed it — but if a learner already completed it before it was hidden, it still counts for them. This is why the same course can behave differently for two learners.
Where to look first. Open the course’s Reports › User statistics, click the learner, and scroll to the material list. The material status icons tell you instantly which one is blocking: green complete · yellow started, not complete · grey not started · padlock locked by a prerequisite.

What Marks Each Material Complete

Section 2

Each material type completes on a different event. If that event never fires, the material — and therefore the course — stays open.

Material typeMarked complete when…
SCORMThe package sends a completion/passed status. The learner must press Close / Exit / Submit inside the content so the status is recorded. Completion rules live inside the package (e.g. all slides viewed, or quiz passed).
xAPI (Tin Can)Depends on the Track all activities setting. On (default): completing the first module completes the whole package. Off: every module must be completed.
VideoReaching 15 seconds before the end (if longer than 150s), or 90% of the way through (if shorter than 150s).

SurveyThe learner clicks Submit.
SlidesThe last slide is viewed.
HTML pageThe learner opens the page.

Why a Course Gets Stuck — Training Material Causes

Section 3

When the on-screen content says “done” but Docebo says In Progress, it’s almost always one of these material-setting causes.

The package never sent completion

The most common case: the learner closed the browser/tab instead of pressing the package’s Close / Exit / Submit button, so SCORM/xAPI never reported back. Session timeouts, dropped connectivity, pop-up blockers, disabled cookies, and Safari are frequent culprits.

The package’s internal completion rule wasn’t met

Completion criteria are set inside the SCORM/Rise/Storyline file (e.g. “all slides viewed” while a free-navigation menu let the learner skip to the end, or “quiz passed” that wasn’t actually passed). Docebo only reflects what the package reports.

xAPI “Track all activities” mismatch

For multi-module / stacked xAPI content, if Track all activities is off, the learner must finish every sub-module. If your sequencing assumes one completion statement, the course hangs after a single module.

No End Object Marker on a mixed course

If the course has optional materials and no end object marker, the course can only complete when all materials are done — including ones you intended as optional.

A required material can never complete by design

An LTI set to require external feedback from a tool that doesn’t send it; a test never submitted; a video the learner scrubbed but didn’t watch to threshold; a file never downloaded. The material type’s trigger simply never fires.

A prerequisite / navigation policy is locking a material

A padlock icon means another material must be completed first. If the prerequisite is itself stuck (or the order is wrong), later materials stay locked and the course can’t finish.

Hidden material that some learners needed

You hid a material to retire it, but learners who had started but not completed it before it was hidden may still be calculated against the old requirement set differently than brand-new learners — producing inconsistent stuck states.

A material was edited after learners started

Modifying any material while a learner is In Progress or Not Started forces them to restart that material from the beginning, which can silently reset progress and re-open the course.

Central Repository shared-tracking surprise

A material reused from the Central Repository shares tracking across every course it appears in. A reset (or incomplete) in one place ripples to others, which can leave a course unexpectedly open.

FAQ & Fixes

Section 4

Click any question to expand. These are ordered roughly from “check this first” to deeper fixes.

Q

A learner says they finished, but the course still shows In Progress. What do I check first?

Find out which material is blocking before changing anything:

  1. Open Course management › the course › Reports tab.
  2. Go to User statistics and click the learner’s name.
  3. Scroll to the material list and find the one that is yellow (started, not complete) or grey (not started).

That material’s type and status tells you which fix below applies. If all materials are green but the course is still open, suspect an End Object Marker / hidden-material edge case or a shared-tracking reset.

Q

The SCORM/Rise content showed a completion screen, but Docebo never registered it. Why?

The package finished but never sent its status back to the LMS. This is a communication break, not usually a config error. Common triggers:

  • The learner closed the tab/window instead of pressing the package’s Close / Exit / Submit button.
  • The browser session timed out mid-course, or connectivity dropped.
  • Pop-up blocker, disabled third-party cookies, or Safari blocking the SCORM calls.
  • Taken on a phone’s mobile browser instead of the Go.learn app.

Fix: have the learner redo it on a desktop/laptop in Chrome with cookies enabled, in a single sitting, pressing Exit at the end — or, if redoing isn’t practical and you’ve verified the package is sound, manually mark them complete (see Admin Actions).

Q

The course has optional materials and I only want one to count. How do I make that work?

Set the material that should finish the course as an End Object Marker in its training material settings. Completing it marks the whole course complete, regardless of the optional materials.

Important: the End Object Marker is not retroactive — it only affects learners going forward. Learners already stuck will need a manual completion or a redo after you set it.

Tip: you can set two markers to build a “test-out” path — pass the first test and you’re done; fail it and you must complete more material and pass a second.

Q

A multi-module xAPI course hangs after the learner finishes a module. What setting causes that?

Check the Track all activities toggle on the xAPI material’s settings (Course management › course › Training material › the xAPI object › Settings).

  • On (default): completing the first module completes the whole package — good for single-statement content.
  • Off: the learner must complete every sub-activity. If they stop after one, the material stays open.

Match the toggle to how the package was authored. Intermittent hangs for only some users usually point to network latency or browser session conflicts rather than the toggle itself.

Q

One material can never be checked off — it stays yellow no matter what. Why?

The material’s completion event isn’t reachable. Match the type to its trigger:

  • LTI set to “after external feedback” but the tool doesn’t send feedback — switch it to complete “when opened,” or fix the tool.
  • Test never submitted, or a minimum passing score not met.
  • Video scrubbed to the end rather than watched to the 90% / last-15-seconds threshold.
  • File opened in preview but never downloaded.

Adjust the material’s completion setting to a reachable trigger, then have the learner redo just that material.

Q

A material shows a padlock and the learner can’t open it. Is that the blocker?

Yes — a padlock means a prerequisite / navigation policy requires an earlier material to be completed first. If the prerequisite is itself stuck, everything after it stays locked and the course can’t finish.

Fix: resolve the prerequisite material first (complete or reset it), or review the course’s navigation policy / prerequisites to confirm the order is what you intend.

Q

I hid a material and now some learners can’t complete. What happened?

Hidden material is not counted toward completion for learners who never viewed it — but it still counts for anyone who completed it before you hid it. That split can leave a subset of learners measured against a different requirement set.

Fix: prefer hiding over deleting (deleting wipes tracking data), and after hiding a material, spot-check stuck learners in User statistics. Use an End Object Marker on a stable material to make completion independent of retired content.

Q

After I edited a material, learners dropped back to In Progress. Is that expected?

Yes. If a learner is In Progress or Not Started and you modify any material in the course, they must restart that material from the beginning. Learners who had already completed the material keep their Completed status.

Fix: make content edits during a maintenance window, communicate to in-flight learners, and avoid editing materials mid-rollout.

Q

How do I just mark the learner complete, and does it fix the material status too?

Manually set the enrollment to Completed: Course management › course › Enrollment tab › change the learner’s status to Completed.

Caveat: this completes the course only. Any material the learner didn’t finish stays marked incomplete in the material-level report. That’s usually fine for a one-off, but it means your material-level analytics won’t reflect a clean finish.

Q

How do I reset a material so the learner can redo it cleanly?

  1. Course Reports › User statistics › click the learner.
  2. Find the material and click the X in the Reset column, then confirm.

Caution: if that material is shared via the Central Repository, the reset applies to every course where it’s used for that learner — check before resetting. (Instructors can always reset tracking for courses they manage.)

Q

Dozens of learners are stuck on the same course. Is there a bulk fix?

First decide whether the root cause is fixable (package rule, Track-all-activities, missing End Object Marker) — fixing the setting and asking learners to redo is cleaner than mass-completing over a real gap.

If a bulk completion is warranted (e.g. a confirmed reporting break on sound content), the LAT toolkit’s bulk_complete.py pattern enrolls and marks a CSV of learners complete with a set date via the Docebo enrollment API — the same approach used to complete 215 users in course 2483 with zero failures. Always confirm the underlying content is sound before bulk-completing, and keep an audit file of who was changed.

Prevention & Best Practices

Section 6
Use one “key” learning object as the End Object Marker. Let it be the single thing that completes the course; track everything else for progress. This makes completion predictable and trivial to troubleshoot.

  • Author USIC content as SCORM 1.2 or SCORM 2004 3rd Edition

    These are the supported, stable standards on USIC Academy. Test the exact package in a sandbox course before publishing, end to end, to confirm it reports completion.

  • Set the package’s internal completion rule deliberately

    In Storyline/Rise, confirm the tracking trigger (e.g. complete when course-complete, not “all slides viewed” on a free-navigation menu). Docebo can only reflect what the package reports.

  • Tell learners how to finish cleanly

    Always press Exit / Close / Submit; use Chrome on a desktop; enable cookies; allow pop-ups; complete in one sitting; on mobile use the Go.learn app. Add a short note to enrollment emails.

  • Break long content into chunks

    Large 1–2 hour packages are more prone to session timeouts and bookmark loss. Smaller modules reduce stuck-completion risk.

  • Freeze content during rollout

    Avoid editing materials while learners are in progress — edits restart that material for anyone not yet complete.
